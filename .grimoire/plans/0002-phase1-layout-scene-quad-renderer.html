Design Decisions

  • SceneItem is a flat Vec, not a tree. RenderNodes already carry the tree structure via Fiber. The scene graph only needs identity + paint order for GPU encoding.
  • Phase 1 Primitive: Quad active, Text/Border/External are stubs. The full enum is defined now to stabilize the scene/ module API; unused variants are no-ops in the renderer.
  • QuadRenderer uses instanced drawing. One 4-vertex unit quad + per-instance buffer (rect, color, corner_radius) → one instanced draw call per batch. This matches the design doc's "instanced quad pipeline" requirement.
  • Layout is a two-pass algorithm. Pass 1: measure intrinsic sizes bottom-up. Pass 2: container calls layout_children() which produces its own Size and child origins.
  • Containers have optional background color. Phase 1 verification needs visible output. Background is a Option<Color> field on container widgets.
  • Widget runtime does its own DP→NDC transform. The QuadRenderer receives a Viewport and converts logical-pixel rects to NDC in the instance buffer, independent of harbor-render's vertex types.
  • SizedBox builder extracted to widgets/. It lived in view.rs during Phase 0 as a test fixture; it now joins the proper widget module.
  • Widgets merge Component + AnyView into one struct. Each widget (Row, Padding, SizedBox, etc.) is a single #[derive(Clone)] struct implementing both Component (pub API via &self) and AnyView (pub(crate) via owned self). build() clones self into the View. This eliminates the XxxState middle layer from Phase 0 — the struct IS the configuration, no separate "state" snapshot needed.

Step 4: Add paint_fiber tree walk in fiber.rs

Files: crates/harbor-widget/src/fiber.rs

Action: Add pub(crate) fn paint_fiber(arena: &FiberArena, id: FiberId) -> Vec<SceneItem>. Walks the fiber tree top-down, accumulates Primitives from each fiber via AnyView::paint_primitives(rect), assigns incrementing paint_order and ids, returns flat Vec<SceneItem>.

pub(crate) fn paint_fiber(arena: &FiberArena, id: FiberId, base_order: u32) -> Vec<SceneItem> {
    let mut items = Vec::new();
    let mut order = base_order;
    let fiber = arena.get(id).unwrap();
    let rect = fiber.layout_rect;
    // Collect self primitives
    if let Some(ref view) = fiber.view {
        if let Some(r) = rect {
            for prim in view.paint_primitives(r) {
                items.push(SceneItem { id: next_scene_id(), primitive: prim, paint_order: order });
                order += 1;
            }
        }
    }
    // Collect children (in order)
    for &child_id in &fiber.children {
        let child_items = paint_fiber(arena, child_id, order);
        order += child_items.len() as u32;
        items.extend(child_items);
    }
    items
}
Outcome: Walking a SizedBox fiber tree produces a Vec of SceneItems. Unit test verifies correct paint order and item count for nested widgets.

Step 6: Add layout_children to AnyView trait and update layout_fiber

Files: crates/harbor-widget/src/view.rs, crates/harbor-widget/src/fiber.rs

Action: Add fn layout_children(&self, constraints: BoxConstraints, child_sizes: &[Size]) -> (Size, Vec<Point>) to AnyView. Default implementation returns (intrinsic_size(constraints), vec![]). For SizedBox, delegate to intrinsic_size with empty positions. Rewrite layout_fiber to be a two-pass algorithm: (1) collect child intrinsic sizes bottom-up, (2) call layout_children on the parent, (3) recurse into children with computed positions.

pub(crate) fn layout_fiber(
    arena: &mut FiberArena,
    id: FiberId,
    constraints: BoxConstraints,
    origin: Point,
) {
    // Phase 1: collect child intrinsic sizes (bottom-up)
    let child_specs: Vec<(FiberId, Size)> = {
        let fiber = arena.get(id).unwrap();
        fiber.children.iter().map(|&cid| {
            let child = arena.get(cid).unwrap();
            // Use loose constraints for intrinsic measurement
            let child_size = child.view.as_ref()
                .map(|v| v.intrinsic_size(constraints))
                .unwrap_or(Size::ZERO);
            (cid, child_size)
        }).collect()
    };

    // Phase 2: compute own size and child origins
    let (own_size, child_origins) = {
        let fiber = arena.get(id).unwrap();
        let sizes: Vec<Size> = child_specs.iter().map(|(_, s)| *s).collect();
        fiber.view.as_ref()
            .map(|v| v.layout_children(constraints, &sizes))
            .unwrap_or((constraints.constrain(Size::ZERO), vec![]))
    };

    // Store own rect
    arena.get_mut(id).unwrap().layout_rect = Some(Rect::from_min_size(origin, own_size));

    // Phase 3: recurse into children with computed positions
    for ((cid, _), child_pos) in child_specs.iter().zip(child_origins.iter()) {
        let child_origin = Point::new(origin.x + child_pos.x, origin.y + child_pos.y);
        let child_constraints = BoxConstraints::loose(own_size);
        layout_fiber(arena, *cid, child_constraints, child_origin);
    }
}
Outcome: Layout works with the new trait method. Existing SizedBox tests pass unchanged.

#[derive(Clone)]
pub struct Padding {
    pub top: f32, pub right: f32, pub bottom: f32, pub left: f32,
    pub background: Option<Color>,
    children: Vec<View>,
}

impl Component for Padding {
    fn build(&self, _cx: &mut BuildCx) -> View {
        View::new(self.clone(), self.children.clone(), None)
    }
}

impl AnyView for Padding {
    fn layout_children(&self, constraints: BoxConstraints, child_sizes: &[Size]) -> (Size, Vec<Point>) {
        let child_size = child_sizes.first().copied().unwrap_or(Size::ZERO);
        let own = Size::new(
            (child_size.width + self.left + self.right).clamp(constraints.min.width, constraints.max.width),
            (child_size.height + self.top + self.bottom).clamp(constraints.min.height, constraints.max.height),
        );
        (own, vec![Point::new(self.left, self.top)])
    }

    fn paint_primitives(&self, rect: Rect) -> Vec<Primitive> {
        match self.background {
            Some(color) => vec![Primitive::Quad { rect, color, corner_radius: 0.0 }],
            None => vec![],
        }
    }
}
Outcome: Padding widget correctly insets and positions its child. Unit test: a SizedBox(100,50) inside Padding(10 each) → layout_rect of 120x70, child at (10,10).

Step 9: Create Row widget (single struct: Component + AnyView)

Files: crates/harbor-widget/src/widgets/row.rs

Action: Create Row struct with fields: cross_axis_alignment: Alignment, background: Option<Color>, children: Vec<View>. Derive Clone. Implement both Component (clone self in build) and AnyView. Layout algorithm: sum child widths, max child height, position children left-to-right with cross-axis alignment.

#[derive(Clone)]
pub struct Row {
    pub cross_axis_alignment: Alignment,
    pub background: Option<Color>,
    children: Vec<View>,
}

impl Component for Row {
    fn build(&self, _cx: &mut BuildCx) -> View {
        View::new(self.clone(), self.children.clone(), None)
    }
}

impl AnyView for Row {
    fn layout_children(&self, constraints: BoxConstraints, child_sizes: &[Size]) -> (Size, Vec<Point>) {
        let total_width: f32 = child_sizes.iter().map(|s| s.width).sum();
        let max_height: f32 = child_sizes.iter().map(|s| s.height).fold(0.0, f32::max);
        let own = constraints.constrain(Size::new(total_width, max_height));
        let mut x = 0.0;
        let positions: Vec<Point> = child_sizes.iter().map(|s| {
            let y = self.cross_axis_alignment.position(s.height, own.height);
            let pos = Point::new(x, y);
            x += s.width;
            pos
        }).collect();
        (own, positions)
    }

    fn paint_primitives(&self, rect: Rect) -> Vec<Primitive> {
        self.background.map(|c| Primitive::Quad { rect, color: c, corner_radius: 0.0 }).into_iter().collect()
    }
}
Outcome: Row correctly stacks children horizontally. Unit test: Row with three 50x50 children → size 150x50, children at x=0,50,100.

Step 14: Create QuadRenderer with instanced quad pipeline

Files: crates/harbor-widget/src/renderer/quad.rs

Action: Create QuadRenderer struct. Constructor creates: (a) vertex buffer with a unit quad [(-0.5,-0.5),(0.5,-0.5),(0.5,0.5),(-0.5,0.5)], (b) index buffer [0,1,2,0,2,3], (c) instanced quad pipeline with vertex layout: position:float32x2 + instance data: rect_ndc:float32x4, color:float32x4, corner_radius:float32. // WGSL vertex shader (quad.wgsl)
struct VertexInput { @location(0) pos: vec2<f32>; }
struct InstanceInput {
    @location(1) rect: vec4<f32>;   // x, y, w, h in NDC
    @location(2) color: vec4<f32>;  // rgba
    @location(3) radius: f32;       // corner radius (unused until Phase 2)
};
struct VertexOutput { @builtin(position) pos: vec4<f32>; @location(0) color: vec4<f32>; }

@vertex
fn vs_main(vert: VertexInput, inst: InstanceInput) -> VertexOutput {
    // transform unit quad into rect
    let x = inst.rect.x + (vert.pos.x + 0.5) * inst.rect.z;
    let y = inst.rect.y + (vert.pos.y + 0.5) * inst.rect.w;
    return VertexOutput(vec4(x, y, 0.0, 1.0), inst.color);
}

@fragment
fn fs_main(in: VertexOutput) -> @location(0) vec4<f32> {
    return in.color;
}

Outcome: QuadRenderer creates pipeline and buffers. Unit test: create with headless device, call update with one Quad, verify instance buffer contents.

Step 15: Wire QuadRenderer into Runtime

Files: crates/harbor-widget/src/runtime.rs

Action: Add renderer: Option<QuadRenderer> field to Runtime. Add pub fn init_renderer(&mut self, device: &wgpu::Device, format: wgpu::TextureFormat) — creates QuadRenderer. Add pub fn encode(&mut self, queue: &wgpu::Queue, pass: &mut wgpu::RenderPass, viewport: Viewport) — applies pending SceneDelta to QuadRenderer, then calls renderer.encode(pass).

impl Runtime {
    pub fn init_renderer(&mut self, device: &wgpu::Device, format: wgpu::TextureFormat) {
        self.renderer = Some(QuadRenderer::new(device, format));
    }

    pub fn encode(&mut self, queue: &wgpu::Queue, pass: &mut wgpu::RenderPass, viewport: Viewport) {
        if let Some(ref mut renderer) = self.renderer {
            // Apply pending delta from last update()
            if let Some(delta) = self.scene_graph.pending_delta() {
                renderer.update(queue, delta, &viewport);
            }
            renderer.encode(pass);
        }
    }
}
Outcome: Runtime::encode() applies SceneDelta to GPU and issues draw calls. Integration test with mocked wgpu verifies no crash on encode.

5 Test Strategy

SceneGraph

Verify diff() correctly identifies added, removed, modified, and unchanged items based on id matching and primitive equality.
Mock None — pure data structure.
Variations
  • Empty → one item (all added)
  • Same items twice (all unchanged, empty delta)
  • Item removed from middle (one removed, later items shift)
  • Item color changes (one modified)
  • Complete clear and rebuild (removed + added)

Padding

Verify layout_children correctly insets child by padding values; intrinsic_size adds padding to child size.
Mock None — pure CPU layout math.
Variations
  • Padding(10,10,10,10) around SizedBox(100,50) → child at (10,10), own 120x70
  • Padding with tight constraints → child clamped to available space
  • Padding(1000,0,0,0) with max constraint 500 → child width 0, padding clipped
  • Asymmetric padding: top=5, right=10, bottom=15, left=20

Row / Column

Verify Children stacked along main axis; cross-axis alignment positions correctly; own size = sum of child extents on main axis, max on cross axis.
Mock None — pure CPU layout math.
Variations
  • Three fixed 50x50 children → Row: 150x50, Column: 50x150
  • Children with different sizes: 100x50 + 200x75 in Row → 300x75
  • CrossAxisAlignment::Center → child centered in cross axis
  • CrossAxisAlignment::End → child at bottom/right edge
  • Empty Row/Column → zero size

Stack / Align

Verify Stack: all children at (0,0), own = max size. Align: child positioned by Alignment within full constraints.
Mock None — pure CPU layout math.
Variations
  • Stack with two 100x50 children → both at (0,0), own 100x50
  • Align::Center in 200x200 with 50x50 child → child at (75, 75)
  • Align with child larger than parent → child clamped by constraints

Viewport

Verify dp_rect_to_ndc produces correct NDC coordinates for a given dp Rect, logical size, and scale factor.
Mock None — pure math.
Variations
  • 100x50 rect at (0,0) in 800x600 @ 1x → NDC (0,0,w,h) verification
  • 2x scale factor doubles pixel dimensions
  • Rect at nonzero origin (50, 30)
  • Zero-size viewport → guard against division by zero

QuadRenderer

Verify Pipeline creation succeeds. Instance buffer update correctly translates SceneDelta items into instance data. encode() produces valid draw calls.
Mock headless wgpu device via wgpu testing utilities; or validate buffer contents directly.
Variations
  • Single Quad → one instance in buffer
  • Multiple Quads → ordered by paint_order
  • Modified Quad (same id, different rect) → instance buffer entry updated
  • Removed Quad → slot marked for reuse or compacted
  • Empty delta → instance count stays at previous value

Runtime (integration)

Verify Update→encode cycle: set_root with widget tree, update produces SceneDelta, encode passes delta to renderer. Idempotent updates produce empty deltas. Viewport changes trigger re-layout.
Mock SceneGraph internal state observable via pending_delta(). No GPU required for layout-only tests.
Variations
  • set_root + update → pending_delta has items
  • Second update with no changes → pending_delta is empty
  • Viewport resize → re-layout, items updated
  • Empty tree → no crash, no items
  • Deeply nested tree (5+ levels) → paint order correct
