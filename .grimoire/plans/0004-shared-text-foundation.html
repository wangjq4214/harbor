Design Decisions

  • Decouple GlyphAtlas from RenderSnapshot. The atlas now accepts &[char] slices. Character collection (scanning dirty ranges, filtering spaces) stays in harbor-render::Text which owns terminal-domain knowledge. This keeps harbor-text free of Harbor domain types beyond harbor-types.
  • Remove cell_width, line_height, ascent from GlyphAtlas. These are used only for vertex building in the GPU layer. They now come from TextMetrics directly, keeping the atlas focused on rasterization and packing.
  • Keep GpuGlyphAtlas and Text in harbor-render. The GPU pipeline, WGSL shader, vertex buffer, and wgpu texture/bind-group management are terminal-specific rendering concerns. The shared text core provides only CPU data.
  • Re-export shared types through harbor-render. pub use harbor_text::{FontBook, load_system_fonts, TextMetrics, AtlasGlyph}; minimizes import churn in the app host. Only the crate manifest and deleted files change visibly.
  • Keep harbor-text dependency on harbor-types minimal. TextMetrics::terminal_size() returns harbor_types::TerminalSize; this is the only coupling point. harbor-types is a zero-dependency crate so the cost is negligible.
  • Depend on harbor-config for constants. FONT_SIZE and TEXT_PADDING are needed by TextMetrics and FontBook. harbor-config is a zero-dependency constants crate — compatible with the "no wgpu, no winit" constraint.

pub struct AtlasUv { pub left: f32, pub top: f32, pub right: f32, pub bottom: f32 }

pub struct AtlasGlyph {
    pub uv: AtlasUv,
    pub width: u32, pub height: u32,
    pub xmin: i32, pub ymin: i32,
    pub atlas_x: u32, pub atlas_y: u32,
}

pub struct RasterizeResult {
    pub new_chars: Vec<char>,
    /// True when the atlas overflowed and was fully rebuilt.
    /// Caller must rebuild all GPU vertices.
    pub evicted: bool,

pub struct GlyphAtlas {
    pixels: Vec<u8>,
    height: u32,
    glyphs: HashMap<char, AtlasGlyph>,
    shelves: Vec<Shelf>,
}

impl GlyphAtlas {
    /// Creates an empty atlas with a zero-filled MAX_ATLAS_SIZE^2 pixel buffer.
    pub fn new() -> Self;

    /// Given a deduplicated & sorted slice of chars, rasterizes any not yet cached.
    /// Returns new chars and eviction flag.
    /// Sort-deduplicates internally if caller hasn't already.
    pub fn rasterize_new(&mut self, fonts: &FontBook, chars: &[char]) -> RasterizeResult;

    /// Drops all cached glyphs and rebuilds from the given unique char set.
    pub fn rebuild(&mut self, fonts: &FontBook, chars: &[char]);

    /// Looks up a cached AtlasGlyph. Returns None if the glyph hasn't been rasterized.
    pub fn glyph(&self, ch: char) -> Option<&AtlasGlyph>;

    /// Number of cached glyphs.
    pub fn len(&self) -> usize;

    /// Access the raw pixel buffer for GPU upload.
    pub fn pixels(&self) -> &[u8];
}

Action detail — char collection moves into Text:

// In Text::prepare_with_dirty — dirty-range char collection
fn collect_unique_chars_from_dirty(
    snap: &RenderSnapshot,
    dirty_ranges: &[DirtyRange],
) -> Vec<char> {
    let mut chars: Vec<char> = dirty_ranges.iter()
        .flat_map(|range| {
            (range.start_col..range.end_col).filter_map(|col| {
                let ch = snap.cell_char(range.row, col);
                if ch != ' ' { Some(ch) } else { None }
            })
        })
        .collect();
    chars.sort_unstable();
    chars.dedup();
    chars
}

// In Text::prepare_with_dirty — atlas update
let unique = Self::collect_unique_chars_from_dirty(snap, dirty_ranges);
let result = self.atlas.rasterize_new(&self.fonts, &unique);
if !result.new_chars.is_empty() {
    self.gpu_atlas.update_glyphs(gpu.queue(), &self.atlas, &result.new_chars);
}
if result.evicted {
    self.dirty = true;
}

Action detail — ensure_glyphs uses new API:

pub fn ensure_glyphs(&mut self, text: &str, _device: &wgpu::Device, queue: &wgpu::Queue) {
    let mut chars: Vec<char> = text.chars().filter(|&c| c != ' ').collect();
    chars.sort_unstable();
    chars.dedup();
    let new_chars = self.atlas.rasterize_new(&self.fonts, &chars).new_chars;
    if !new_chars.is_empty() {
        self.gpu_atlas.update_glyphs(queue, &self.atlas, &new_chars);
    }
}

4 Edge Cases

#ConditionExpected BehaviorOwning Step
1 Atlas overflows (new shelf would exceed MAX_ATLAS_SIZE) during incremental update in rasterize_new GlyphAtlas triggers an internal full rebuild, clears all shelves and pixels, and returns RasterizeResult { evicted: true, new_chars: all_chars }. The caller in Text::prepare_with_dirty sets self.dirty = true, causing a full vertex buffer rebuild with the new UVs. Step 7
2 Empty char slice passed to rasterize_new or rebuild Returns RasterizeResult { new_chars: vec![], evicted: false } immediately with no allocation. The caller skips GPU upload and vertex rebuild. Step 4
3 Char slice contains only space characters after filtering The caller (Text) is responsible for filtering spaces before calling rasterize_new. If spaces leak through (e.g., ' ' is explicitly passed), fontdue::Font::rasterize(' ', px) returns a zero-width glyph which would create a degenerate shelf entry. GlyphAtlas skips chars where the rasterized glyph width is 0. Step 7
4 Text resize (grid rows/cols change) while atlas has cached glyphs Text::prepare_with_dirty detects dimension mismatch, calls atlas.rebuild(fonts, &all_chars) to flush stale UVs, recreates the GPU atlas texture, reallocates the vertex buffer if capacity increased, and rebuilds all vertices. The old pixel data is replaced; the persistent glyph-to-UV cache is only as large as the new visible character set. Step 7
5 Font loading fails entirely (no system fonts, empty fontdb) load_system_fonts() returns Err(anyhow!("no system fonts found")). The app fails to start with a clear error — unchanged from current behavior. Step 2
6 CJK probe char not found in any font (primary has CJK but no fallback is needed, or fallback loading fails) FontBook is created with only the primary font. CJK chars rasterize as tofu (empty glyph) — the same degradation behavior as today. A warning is logged. Step 2
