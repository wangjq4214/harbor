Design Decisions

  • View uses type-erased Box<dyn AnyView> — enables open widget set without modifying View for each new widget type.
  • FiberArena wraps slotmap::SlotMap — leverages a proven generation-checked arena instead of hand-rolling index+generation pairs.
  • Signal<T> uses pull-based version tracking with dirty-flag notification — aligns with the design document's explicit dirty-flag taxonomy (BUILD_DIRTY, LAYOUT_DIRTY, etc).
  • BuildCx holds a direct FiberId reference for hook addressing — simpler and more predictable than thread-local context.
  • Geometry types are self-contained with no external math crate — Phase 0 needs only Point, Size, Rect, BoxConstraints; no transforms or complex geometry.
  • Component trait has a single build method — lifecycle hooks (mount, unmount) belong to Fiber management, not user code.
  • Top-left coordinate origin — matches screen/UI convention (wgpu, Flutter, SwiftUI, browsers); avoids confusion when RenderNodes later map to GPU scissor rects.

Step 3: Implement BoxConstraints

Files: crates/harbor-widget/src/layout/constraints.rs

Action:

pub struct BoxConstraints { pub min: Size, pub max: Size }

impl BoxConstraints {
    pub fn tight(size: Size) -> Self { /* min == max == size */ }
    pub fn loose(max: Size) -> Self { /* min = zero, max = max */ }
    pub fn constrain(&self, size: Size) -> Size {
        Size::new(
            size.width.clamp(self.min.width, self.max.width),
            size.height.clamp(self.min.height, self.max.height),
        )
    }
    pub fn is_tight(&self) -> bool { self.min == self.max }
}
Outcome: constrain clamps correctly; tight(Size::new(100, 50)) produces min == max.

Group E: Fiber Tree

Step 6: Implement DirtyFlags, FiberId, Fiber, FiberArena

Files: crates/harbor-widget/src/fiber.rs

Action:

use bitflags::bitflags;
use slotmap::{SlotMap, new_key_type};

new_key_type! {
    pub struct FiberId;
}

bitflags! {
    pub struct DirtyFlags: u8 {
        const BUILD_DIRTY    = 0b0001;
        const LAYOUT_DIRTY   = 0b0010;
        const PAINT_DIRTY    = 0b0100;
        const HIT_TEST_DIRTY = 0b1000;
    }
}

pub struct Fiber {
    pub id: FiberId,
    pub key: Option<Key>,
    pub widget_type: TypeId,
    pub hooks: Vec<Box<dyn Any>>,  // type-erased Signal<T> values
    pub children: Vec<FiberId>,
    pub parent: Option<FiberId>,
    pub flags: DirtyFlags,
    pub layout_rect: Option<Rect>,
}

pub struct FiberArena {
    fibers: SlotMap<FiberId, Fiber>,
}
impl FiberArena {
    pub fn insert(&mut self, fiber: Fiber) -> FiberId { self.fibers.insert(fiber) }
    pub fn get(&self, id: FiberId) -> Option<&Fiber> { self.fibers.get(id) }
    pub fn get_mut(&mut self, id: FiberId) -> Option<&mut Fiber> { self.fibers.get_mut(id) }
    pub fn remove(&mut self, id: FiberId) -> Option<Fiber> { self.fibers.remove(id) }
    pub fn contains(&self, id: FiberId) -> bool { self.fibers.contains_key(id) }
}

Add slotmap to Cargo.toml deps. Add bitflags or hand-roll DirtyFlags bit ops (hand-roll preferred to avoid extra dep).

Outcome: Fibers inserted into arena; retrieved by FiberId; stale IDs (generation mismatch) return None; DirtyFlags bit operations correct.

Group F: Reconciliation

Step 7: Implement reconciliation logic on FiberArena

Files: crates/harbor-widget/src/fiber.rs

Action: Add reconcile method to FiberArena (or as a free function):

impl FiberArena {
    /// Reconcile a child list against new Views. Returns (new_child_ids, dirty flags set).
    pub fn reconcile_children(
        &mut self,
        parent_id: FiberId,
        old_children: &[FiberId],
        new_views: Vec<View>,
        cx: &mut BuildCx,
    ) -> Vec<FiberId>;
}

Algorithm matches design doc rules:

  1. Walk both old children and new views by position.
  2. If same position, same widget_type(), same key() → reuse Fiber (preserve hooks, state).
  3. If type or key differs → unmount old subtree (clear subscriptions, remove children recursively), then create new Fiber via View::into_inner().build(cx).
  4. No-key items match by position only.
  5. Unmount on mismatch: walk children recursively, unsubscribe from Signals, remove from arena.
Outcome: Same type+key → hook state preserved across rebuilds. Key change → old fiber destroyed, new created. Stale FiberId rejected.

Group G: Runtime Assembly

Step 8: Implement Runtime with set_root, update, dirty queue

Files: crates/harbor-widget/src/runtime.rs

Action:

pub struct Runtime {
    arena: FiberArena,
    root_id: Option<FiberId>,
    dirty_queue: Vec<FiberId>,
}

impl Runtime {
    pub fn new() -> Self { /* empty */ }

    pub fn set_root(&mut self, root: impl Component + 'static) {
        // 1. Build initial View from component.
        // 2. If root_id exists, unmount old root subtree.
        // 3. Reconcile: create new Fiber from View.
        // 4. Set root_id.
        // 5. Mark root BUILD_DIRTY | LAYOUT_DIRTY.
        // 6. Push root_id to dirty queue.
    }

    pub fn update(&mut self, _now: Instant) -> FrameRequest {
        // Process dirty queue:
        //   1. Pop each dirty fiber.
        //   2. If BUILD_DIRTY: build component → reconcile children.
        //   3. If LAYOUT_DIRTY: run layout pass from this fiber down.
        //   4. Clear flags.
        // Return FrameRequest { needs_redraw: !dirty_queue.is_empty() } in Phase 0.
    }
}

pub struct FrameRequest {
    pub needs_redraw: bool,
}

Layout pass in Phase 0: walk the tree top-down, pass BoxConstraints to each node, each node computes its Rect and stores in Fiber.layout_rect. For now, a default tight constraint flow.

Outcome: set_root + update produces a Fiber tree with layout rects. Empty tree → no crash. Multiple update calls without changes → no re-layout (dirty queue empty).

5 Test Strategy

Signal<T>

VerifyValue reads correctly after set; version increments; subscribers added/removed; set with no subscribers is a no-op (no panic).
MockFiberId uses SlotMap's DefaultKey — no mock needed; FiberId created from a test arena.
Variations
  • Initial value accessible via read()
  • set() updates value and increments version
  • Multiple subscribers receive dirty notification
  • Unsubscribed fiber not notified on subsequent sets
  • Same fiber subscribed twice → idempotent

BoxConstraints

Verifytight() produces min == max; loose() produces min == zero; constrain clamps within bounds.
MockNone — pure data, no dependencies.
Variations
  • Size within bounds → returned unchanged
  • Size below min → clamped to min
  • Size above max → clamped to max
  • min > max in one axis → constrain uses min (no panic)
  • Zero-size constraints

Rect / Point / Size

VerifyRect::from_min_size correct; Rect::size accurate; Rect::contains works for inside/boundary/outside points.
MockNone — pure data.
Variations
  • Zero-size rect (min == max)
  • Point at rect boundary → contains is true
  • Point outside rect → contains is false

FiberArena

VerifyInsert returns valid FiberId; get with valid id returns Fiber; get with stale id (removed slot) returns None; remove returns Fiber and invalidates id.
MockNone — SlotMap behavior is deterministic and fast.
Variations
  • Insert → get → remove → get (stale) returns None
  • Bulk insert and iteration
  • Removing a parent does not implicitly remove children (caller responsibility)

Reconciliation (fiber.rs)

VerifySame type+key → Fiber reused, hooks preserved. Key change → old unmounted, new created. Type change → same. Position match for no-key items. Nested children reconcile correctly.
MockTest widgets with predictable View output. Signal subscriptions can be verified by checking subscriber count after unmount.
Variations
  • Static tree (no changes) → all fibers reused, no new hooks
  • Key change at position 1 of 3 → only position 1 Fiber replaced
  • Widget type change → old unmounted, new created
  • Child list grows (3 → 5) → old 3 reused, 2 new created
  • Child list shrinks (5 → 2) → first 2 reused, last 3 unmounted
  • Unmount clears Signal subscriptions

Runtime

Verifyset_root initializes tree; update processes dirty queue; FrameRequest.needs_redraw is true after first build, false on subsequent empty update; multiple set_root calls don't leak.
MockUse SizedBox as concrete Component. No GPU/external deps.
Variations
  • set_root → update (needs_redraw = true) → update (needs_redraw = false)
  • set_root twice → second call unmounts first tree
  • Signal write → update → fiber rebuilt
  • Nested SizedBoxes produce correct layout rects

BuildCx + use_state

VerifyFirst call to use_state creates Signal; second call on same Fiber returns existing Signal; hook order is stable across rebuilds.
MockFiber with pre-allocated hooks slot; BuildCx with known FiberId.
Variations
  • Single use_state → returned Signal persists across rebuilds
  • Multiple use_state calls → order-preserving, each returns distinct Signal
  • Conditional use_state (called in one build, skipped in next) → hook index mismatch; Phase 0 can panic (consistent with React rules of hooks)
