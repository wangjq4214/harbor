Design Decisions

  • Event handlers on AnyView, not Fiber callback table. Keeps widget behavior local to the widget struct. Avoids introducing a generic callback registration system for a single use case (Button). Consistent with existing paint_primitives / layout_children pattern.
  • InputState extracted from Runtime. Focus/hover/capture state has its own lifecycle (mutated by events) separate from Runtime's scheduling lifecycle. Independently testable without a full Runtime.
  • FocusScope as widget, not separate tree. Participates in layout and paint — zero special paths. Modal interception is a capture-phase early-exit check against FocusScope.modal.
  • EventHandled vs StopPropagation are distinct. EventHandled::Handled signals "I consumed this event" (for future potential use). StopPropagation in EventCtx forces the walk to stop regardless of Handled status. Follows DOM event model precedent.
  • Hit test uses reverse paint order (DFS, children reversed). Matches design doc §8 specification. Stack children painted later (on top) get first chance at hit testing.
  • Modal blocks at capture phase, before target phase. If a modal FocusScope is in the ancestor chain and the target is not in its subtree, the event is silently consumed. This blocks keyboard events to the terminal while allowing modal buttons to work.

pub(crate) trait AnyView: 'static {
    // ... existing methods ...

    /// Returns true if point (in local coordinates) is inside the widget.
    /// Default: rect.contains(point).
    fn hit_test(&self, point: Point, rect: Rect) -> bool {
        rect.contains(point)
    }

    /// Handles an input event. Returns EventHandled::Handled if consumed.
    /// Default: ignores all events.
    fn handle_event(&self, event: &UiEvent, ctx: &mut EventCtx, rect: Rect) -> EventHandled {
        let _ = (event, ctx, rect);
        EventHandled::Ignored
    }

    /// Whether this widget can receive focus via Tab navigation.
    /// Default: false.

        false
    }
}

Step 5: Implement hit testing and event routing in Runtime

Files: crates/harbor-widget/src/runtime.rs (modify)

Action: Add input: InputState to Runtime::new(). Add private hit_test and route_event methods. Add public dispatch method.

// Hit test: reverse paint order DFS walk
fn hit_test(&self, fiber_id: FiberId, point: Point) -> Option<FiberId> {
    let fiber = self.arena.get(fiber_id)?;
    let rect = fiber.layout_rect?;
    if !rect.contains(point) { return None; }

    // Children in reverse order (topmost first)
    for &child_id in fiber.children.iter().rev() {
        if let Some(hit) = self.hit_test(child_id, point) {
            return Some(hit);
        }
    }

    // Check self
    let local_point = Point::new(point.x - rect.min.x, point.y - rect.min.y);
    if fiber.view.as_ref().map_or(false, |v| v.hit_test(local_point, rect)) {
        return Some(fiber_id);
    }
    None
}

// Event routing: capture → target → bubble → apply commands
fn route_event(&mut self, event: &UiEvent) -> bool {
    let Some(root_id) = self.root_id else { return false; };

    // 1. Determine target
    let target = match event {
        UiEvent::Pointer(pe) => {
            self.input.pointer_captures.get(&pe.pointer_id).copied()
                .or_else(|| self.hit_test(root_id, pe.position))
        }
        UiEvent::Keyboard(_) | UiEvent::Focus(_) => self.input.focused,
    };

    // 2. Build ancestor path
    let path = self.build_ancestor_path(target);

    // 3. Run capture → target → bubble
    let mut ctx = EventCtx::new();
    self.walk_capture(&path, target, event, &mut ctx);
    if !ctx.is_propagation_stopped() {
        self.walk_target(target, event, &mut ctx);
    }
    if !ctx.is_propagation_stopped() {
        self.walk_bubble(&path, target, event, &mut ctx);
    }

    // 4. Apply commands
    let needs_paint = self.input.apply(ctx.take_commands()) || ctx.needs_paint();
    needs_paint
}

pub fn dispatch(&mut self, event: UiEvent, _now: Instant) -> FrameRequest {
    let needs_redraw = self.route_event(&event);
    if needs_redraw {
        if let Some(root_id) = self.root_id {
            crate::signal::mark_dirty(root_id);
        }
    }
    FrameRequest { needs_redraw }
}
Outcome: Events route through the tree correctly. Hit test finds topmost widget. Pointer capture bypasses hit test. Modal blocks events. Commands applied after walk.

Step 7: Implement Button widget

Files: crates/harbor-widget/src/widgets/button.rs (create)

Action: Implement Button as Component + AnyView. Store label, onClick closure, and visual state. Override handle_event for pointer/focus. Override is_focusable to return true. Paint primitives for background with state-dependent color.

pub struct Button {
    label: String,
    on_click: Option<Box<dyn Fn(&mut EventCtx)>>,
    // Visual state tracked here, mutated via handle_event
    state: ButtonVisualState, // Normal, Hovered, Pressed, Focused
}

impl AnyView for Button {
    fn is_focusable(&self) -> bool { true }

    fn handle_event(&self, event: &UiEvent, ctx: &mut EventCtx, _rect: Rect) -> EventHandled {
        match event {
            UiEvent::Pointer(PointerEvent { phase: PointerPhase::Up, .. }) => {
                if let Some(ref cb) = self.on_click { cb(ctx); }
                EventHandled::Handled
            }
            UiEvent::Pointer(PointerEvent { phase: PointerPhase::Down, .. }) => {
                ctx.capture_pointer(event.pointer_id());
                EventHandled::Handled
            }
            UiEvent::Keyboard(KeyboardEvent::KeyDown { key: Key::Enter | Key::Space, .. }) => {
                if let Some(ref cb) = self.on_click { cb(ctx); }
                EventHandled::Handled
            }
            _ => EventHandled::Ignored,
        }
    }

    fn paint_primitives(&self, rect: Rect) -> Vec<Primitive> {
        let color = self.state.background_color();
        vec![Primitive::Quad { rect, color, corner_radius: 4.0 }]
    }
}
Outcome: Clickable, focusable button with visual state. Sufficient for paste confirmation confirm/cancel.

Step 8: Wire modal interception into event routing

Files: crates/harbor-widget/src/runtime.rs (modify)

Action: During capture phase walk, check each ancestor: if it's a FocusScope with modal: true and the target is not in its subtree, stop the event walk.

fn is_descendant_of(&self, descendant: Option<FiberId>, ancestor: FiberId) -> bool {
    let mut current = descendant;
    while let Some(id) = current {
        if id == ancestor { return true; }
        current = self.arena.get(id).and_then(|f| f.parent);
    }
    false
}

// In walk_capture:
for &ancestor_id in &path {
    // Check modal barrier

            if view.is_modal_scope() && !self.is_descendant_of(target, ancestor_id) {
                return; // Event blocked by modal
            }
